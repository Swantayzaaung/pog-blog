What's Pogblog?

Pogblog is a simple (or more frankly put, rudimentary) website I made for my Final Project for the CS50 Introduction to Computer Science course, an awesome, free computer science course made by Harvard. As of now, I've only added the basic posting feature so that users can post their stores in a text-only format, although if I have the time, I'll make sure to add more of the essential features of many blogging sites, including likes, comments, saving, communities, etc. So I'm really sorry if this site feels like a complete desert right now, but I gotta submit it first, then do those extra features afterwards.


Why I created Pogblog

For my Final Project, I kept struggling to get an idea of what to make for it. I didn't want it to be too simple, or it wouldn't be worth the course. Nor too complicated, which would end up taking way too much time for my schoolwork and other stuff. At first I decided to create a memeing website, where you could search up meme templates and edit image macros and memes. But when I actually started with the job, I found out that endlessly looking for meme template APIs and creating a raster graphics editor was nigh impossible to finish within a short time. Soon, I gave up with that, then went for an alien shooter game using Pygame, which went pretty well in my opinion, but would still take too long.

So finally, I decided to just go a simpler route - creating a blogging site. It was sufficiently complex , with the tiny details such as layouts and data validation taking some time to figure out, but not too large and complicated to the point it'd take months for me to develop. Still, I ended up taking about two weeks just to finish this skeleton of a website.


Technologies used here

Another reason I went for this kinda project is that it applies the concepts and technologies taught at CS50. Poglog is a Flask based web application, using the Flask library in Python for the application, in conjunction with a SQLite3 database for storing user and post data, and HTML, CSS and JavaScript for the frontend templates. Working on this project helped me deepen my understanding of the various components of Flask and SQL, as well as learning to use data validation in JavaScript, something I've never tried before. Admittedly, I took way longer than I expected on the layout part, often taking hours to get some simple divs to display side by side. Don't get me started on the navbar part either, or the endless battle for font sizes and paddings with Bossy Ol' Bootstrap (which I later decided to not use in favour of a custom navbar style).
